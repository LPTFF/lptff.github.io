## 二、怎么背：项目 → 原理 → 八股

不要从「知识树顶层」往下背。  
要建立：**真实项目问题 → 你怎么做 → 追问落到哪条八股**。

### 万能答题结构（30 秒 + 2 分钟）

1. **场景**：什么业务、什么页面、什么指标（首屏、卡顿、权限漏放）
2. **手段**：你/团队做了什么（可量化）
3. **原理**：为什么有效（对应下面链路里的节点）
4. **权衡**：代价、边界、如果重来会怎么改

### 四条主链路（覆盖你 80% 面试）

每个节点括号内为原稿检索关键词（去 `前端八股文汇总背诵版.md` 搜）。

### 链路 A：Agent 页面 / 探索型前台 — 「为什么慢？怎么优化？」

```
Agent 对话页 / 多模块首屏慢、交互卡顿
    ↓ 现象与指标
LCP / 白屏 / TTI 变差（Performance API、Core Web Vitals）
    ↓ 加载与构建
路由懒加载、异步组件、Webpack/Vite 分包（Code Splitting、dynamic import、splitChunks）
    ↓ 缓存
强缓存 / 协商缓存、静态资源 hash、CDN（HTTP 缓存、Cache-Control、ETag）
    ↓ 运行时
减少首屏 JS、Tree-shaking、按需引入（Webpack 优化、sideEffects）
    ↓ Vue 更新
异步组件挂载顺序、nextTick 等 DOM 更新后再测量/操作
    ↓ 渲染
虚拟 DOM Diff 只更新变化子树；避免无效父组件重渲染（Diff、key、memo 思路）
    ↓ 异步与调度
请求链、Promise 链、微任务里更新 UI（事件循环、宏任务/微任务）
    ↓ 体验兜底
骨架屏、防抖（输入）、节流（滚动）（防抖节流）
```

**必背八股节点**：事件循环 · Promise · nextTick · Diff · 缓存 · Webpack 分包 · 首屏优化  

**可讲的项目话术示例**：  
「Agent 页模块多，我们把非首屏路由改成 lazy + prefetch 降级，构建侧拆 vendor 和 agent 业务 chunk，配合协商缓存；交互侧输入用防抖，流式输出用 nextTick 合并 DOM 更新，避免同一 tick 多次 patch。」

**原稿索引**：`# 前端性能优化` · `# webpack` · `# Vite` · `# Vue`（nextTick、异步）· `# 浏览器`（缓存、渲染）· `# HTTP`（缓存头）

### 链路 B：内部交易系统 / 行情列表 — 「大列表、高频刷新怎么扛？」

```
订单簿 / 行情表格：上千行、秒级刷新
    ↓ 现象
滚动掉帧、内存涨、选中态丢失
    ↓ 方案
虚拟列表（只渲染可视区 + 缓冲区）
    ↓ DOM
减少回流：固定行高、transform 动画、避免逐项改 layout（回流重绘）
    ↓ Vue
v-for + 稳定 key；子组件拆分；必要时 Object.freeze 冻结纯展示数据
    ↓ 更新策略
computed 缓存派生列；watch 精确依赖；避免深度 watch 大对象
    ↓ 通信
父传子 props + 事件回调；跨模块用 provide/inject 或 Pinia 按域拆分
    ↓ 性能兜底
节流合并推送（如 100ms 一批）；WebSocket 与轮询选型（WebSocket、事件循环）
    ↓ 工程
打包不把整表组件打进首屏 chunk；Worker 处理重计算（可选，了解即可）
```

**必背八股节点**：大列表 · 回流重绘 · computed/watch · key · 组件通信 · 防抖节流  

**原稿索引**：`# Vue`（v-for/key、computed/watch、通信）· `# CSS`（回流重绘）· `# JS`（事件循环）· `# 前端性能优化`（长任务、虚拟列表）

### 链路 C：金融系统 — 「权限、安全、合规」

```
菜单 / 按钮 / 接口三级权限
    ↓ 前端
路由守卫 + 动态路由；按钮 v-if / 自定义指令；路由 meta.roles
    ↓ 不能只做前端
Token / Session；HTTPS；敏感操作二次校验（HTTPS、Cookie、JWT）
    ↓ 攻击面
XSS（v-html、富文本）、CSRF（SameSite、Token）、越权（前端权限设计）
    ↓ 跨域与部署
开发 proxy、生产 Nginx 反向代理；CORS 简单请求与预检（跨域）
    ↓ 状态
登录态存哪：Cookie HttpOnly vs 内存 + 短 Token；刷新 Token 流程
    ↓ 负责人视角
权限模型可配置、可审计；与后端 RBAC 对齐；埋点不记录敏感字段（权限、埋点）
```

**必背八股节点**：权限设计 · HTTPS · 跨域 · XSS/CSRF · 路由守卫  

**原稿索引**：`# 其他`（权限、安全）· `# Vue`（路由守卫）· `# 浏览器`（存储、同源）· `# HTTP`（HTTPS、CORS）

### 链路 D：埋点 / 监控 / 稳定性 — 「怎么知道线上好不好？」

```
线上错误率、接口慢、白屏
    ↓ 采集
window.onerror、unhandledrejection、Vue errorHandler、axios 拦截器
    ↓ 上报
批量、节流、采样；sendBeacon；不阻塞主线程（事件循环、微任务）
    ↓ 性能
Performance API、LCP/FCP、资源 Timing（Performance、Core Web Vitals）
    ↓ 工程
Source Map 反解、环境区分、release 版本号（Webpack、环境变量）
    ↓ 闭环
告警 → 定位 chunk / 接口 → 回滚或热修
```

**必背八股节点**：埋点思路 · Promise/异步 · 性能指标 · 错误边界（Vue 项目可提 errorHandler）  

**原稿索引**：`# 其他`（监控）· `# JS`（Promise、事件循环）· `# 前端性能优化`

### 链路 E：JS 基础 — 「手写 / 原理追问」— 独立贯通链

任何项目都可能突然切到：

```
事件循环（宏任务/微任务执行顺序）
    ↔ Promise / async-await（状态、链式 then）
    ↔ 闭包（定时器、防抖闭包存 state）
    ↔ this（箭头函数、call/apply/bind）
    ↔ 原型链（instanceof、继承）
    ↔ 深浅拷贝（嵌套对象、循环引用）
    ↔ 防抖节流（搜索、resize、滚动）
```

**建议**：每条链路至少准备 **1 道手写 + 1 道输出题**。  
原稿：`# JS` · `# 在线笔试题`（事件循环、Promise、Foo.getName 类题）

---

### 链路 F：浏览器 + 网络 — 「从输入 URL 到页面出来」

```
输入 URL → 回车
    ↓ DNS 解析 → TCP 三次握手 → TLS（HTTPS）
    ↓ 请求报文（GET/POST、Header、Cookie）
    ↓ 响应（状态码、缓存是否命中 200/304）
    ↓ 解析 HTML → 构建 DOM；CSS → CSSOM → 渲染树
    ↓ JS 阻塞解析？defer/async？
    ↓ 布局 Layout → 绘制 Paint → 合成 Composite
    ↓ 回流重绘触发条件与优化
```

**必背八股节点**：输入 URL · TCP/HTTPS · 缓存 · 渲染过程 · 回流重绘  

**原稿索引**：`# 浏览器` · `# HTTP`

## 四、场景题答题模板（面试直接用）

### 1. 首屏优化

- **指标**：FCP、LCP、TTI（各是什么）
- **加载**：路由懒加载、分包、preload 关键资源、CDN、HTTP/2
- **缓存**：hash 文件名 + 长缓存 + index 短缓存
- **渲染**：减 JS、SSR/骨架屏（若做过）、关键 CSS
- **Vue**：异步组件、避免首屏过大 store、nextTick 后测速
- **串联**：链路 A

### 2. 权限

- **模型**：RBAC、路由级 + 按钮级 + 接口级
- **实现**：动态路由、守卫、meta、指令、后端二次校验
- **安全**：Token 存储、XSS 导致权限绕过、越权
- **串联**：链路 C

### 3. 埋点

- **类型**：曝光、点击、性能、错误
- **实现**：统一 SDK、队列批量、采样、sendBeacon
- **合规**：脱敏、金融场景最小采集
- **串联**：链路 D

### 4. 大列表

- **虚拟滚动**原理；固定行高 vs 动态高度
- **Vue**：key、拆分组件、冻结数据、节流更新
- **CSS**：contain、避免强制同步布局
- **串联**：链路 B

### 5. 技术负责人向加答

- 方案评审维度：性能、安全、可维护、排期风险
- 规范：ESLint、Code Review、上线 checklist
- 带人：难点拆解、技术债怎么还
